    AI Proxy
    Not configured
    The "Identify with AI" scan feature needs to reach Anthropic's API. Browsers block direct calls, so point this at your own Cloudflare Worker (or similar proxy) that holds your API key server-side. See the setup guide below.
    Cloudflare Worker Setup (free tier)
    1. Sign in at dash.cloudflare.com (free account is fine)
    2. Left sidebar → Compute (Workers) (may appear under "Compute & AI" depending on your dashboard). Click Workers & Pages.
    3. Click Create (or Create application) → Start with Hello World! → Get started
    4. Name it card-vault-proxy → Deploy. Cloudflare creates a basic worker and gives you a .workers.dev URL.
    5. After deployment, click Continue to project (or Edit code from the worker's overview page).
    6. In the code editor, select all existing code in worker.js (or index.js), delete it, then paste the code from the "Show worker.js Code" button below. Click Deploy in the top right.
    7. Go back to the worker overview → Settings tab → scroll to Variables and Secrets → click Add:
      • Type: Secret
      • Variable name: ANTHROPIC_API_KEY
      • Value: your API key from console.anthropic.com → Settings → API Keys
      • Click Deploy.
    8. (Recommended) Click Add again:
      • Type: Secret
      • Variable name: ALLOWED_ORIGIN
      • Value: your GitHub Pages URL root, e.g. https://yourname.github.io (no trailing slash, no path)
      • Click Deploy. This locks the proxy to only your app.
    9. On the worker overview page, copy the URL shown near the top (ends in .workers.dev). Paste it into the Proxy URL field above, click Save, then Test Connection. You should see "✓ Proxy reachable".
    Troubleshooting: If "Add" is missing under Variables and Secrets, make sure you clicked into your worker first (not the Workers & Pages landing page). If Test Connection fails with CORS errors, double-check ALLOWED_ORIGIN matches your GitHub Pages URL exactly.
